3. Compatibility Checks

Need to test three things separately
- plain logit lens compatibility
- tuned-lens compatibility
- J-lens compatibility

Plain logit lens compatibility
- HF model loads
- tokenizer loads
- final norm and output head accessible
- hidden states available
- one small batch runs

Tuned-lens compatibility
- TunedLens.from_model(hf_model) works for architecture smoke test
- trained tuned lens loads later with from_model_and_pretrained(...)
- one small batch runs and emits logits per layer

J-lens compatibility
- jlens.from_hf(hf_model, tokenizer) works
- one small fit run starts on exported train_fit prompts
- if fit starts without architecture error, treat model as fit-compatible
- if checkpoint is created, inspect n_done / source_layers / next_idx

Compatibility report per model
- outputs/<family>/compatibility/<model_label>_compatibility.json

5. Batch Capture

Need one unified long-form capture format for all models and lenses.

Always capture
- final model logits
- plain logit-lens logits
- tuned-lens logits if available
- J-lens logits if available
- hidden states needed for downstream hidden-space analysis
- enough metadata to reconnect tensor artifacts to flat analysis rows

Need two storage layers

Layer A: flat long-form analysis rows
- metrics computed on the fly
- top-k ids
- top-k decoded tokens
- source token text
- target token text
- final top-1 token
- lens top-1 token
- references to saved tensor artifacts

Layer B: reusable tensor artifacts
- final model logits
- logit-lens logits
- tuned-lens logits if available
- J-lens logits if available
- hidden states / residuals needed for hidden-space comparisons

Tensor artifact strategy
- save tensors at prompt granularity by default so each prompt has its own reusable artifact file
- prompt-level artifacts are easier to resume, inspect, debug, and rerun selectively
- each artifact file gets a stable artifact_id
- flat rows store artifact_id plus indices needed to recover:
  - example index
  - layer index
  - position index
  - lens name where relevant
- use a project-owned format such as:
  - .pt for exact PyTorch tensors
  - optional lighter derived exports later if needed

Minimum tensor artifacts to preserve
- final logits
- per-lens logits
- final hidden state / final residual
- intermediate hidden states or residuals used for plain logit lens
- J-lens transported residuals if needed for hidden-space comparison
- tuned-lens transformed hidden states if needed for hidden-space comparison

Reason
- some downstream analyses need the full logits, not just summary metrics
- hidden-space metrics require access to actual hidden states / residuals
- saving both tensor artifacts and flat summaries avoids recomputing expensive runs

Metric computation rules
- distinguish clearly between:
  - hidden-space quantities
  - raw logits
  - probability distributions after softmax
- any metric defined on probabilities must be computed from softmaxed logits, not from hidden states
- any hidden-space metric must explicitly state which hidden state / residual variant it uses
- preserve enough artifacts to recompute metrics later if needed

Probability rules
- softmax over vocab dimension only
- use float32 for probability computations even if model weights are bfloat16 / float16
- check probability sums are numerically close to 1.0
- clamp only where mathematically necessary for logs / KL, and record epsilon used

Metric validation and assertions
- every metric pipeline should include assertions or validation checks
- fail fast if a metric is outside its valid range unless the metric is explicitly unbounded

Metric taxonomy

Intrinsic single-distribution metrics
- compute these for every saved prediction distribution:
  - logit mean
  - logit std
  - logit min
  - logit max
  - logit top1_top2_gap
  - probability mean
  - probability std
  - entropy
  - effective support / perplexity-from-probs
  - top-k mass
  - top-1 margin
  - skewness
  - kurtosis

Pairwise distribution metrics
- same model, different lenses
- same lens, different models
- examples:
  - KL(P || Q)
  - KL(Q || P)
  - JS(P, Q)
  - TV(P, Q)
  - top-k Jaccard
  - gold-token probability difference
  - final-top1 probability difference
  - entropy difference
  - top-1 margin difference

Hidden-space metrics
- cosine similarity
- L2
- relative L2
- norm ratio
- any additional residual-space distances we later decide to keep

Fuzzy-trace style metric groups
- keep the fuzzy-trace framing from the earlier notebooks
- separate metrics into:
  - verbatim / exact-token fidelity
  - gist / surface-form proxy fidelity
  - mechanistic / hidden-space faithfulness

Verbatim / exact-token metrics
- top1_exact
- topk_exact
- gold_rank
- gold_prob
- final_top1_rank

- gt_prob_diff

Gist / surface-form proxy metrics

Mechanistic / hidden-space metrics

- L2 to final residual
- relative L2 to final residual
- norm ratio to final residual
- any later residual-space metrics should state exactly which residual variants are compared

Notebook-derived experimental metrics to preserve
- the line-plot metrics from the earlier exploratory notebooks should all remain capturable
- especially:
  - gold_rank
  - gold_prob
  - gt_prob_diff
  - top-k Jaccard

Hidden-state artifact requirements

- for plain logit lens:
  - save intermediate residuals before final unembedding
- for J-lens:
  - save both source residuals and transported residuals when needed
- for tuned lens:
  - save pre-translation hidden states and translated hidden states when needed
- always save enough metadata to know which hidden-state variant a metric used

Numerical support checks
- check for NaN / inf in logits before metric computation
- check for NaN / inf after softmax
- check tensor shapes before saving
- assert expected vocab size agreement across final logits and lens logits
- assert expected layer count where applicable

Capture granularity rules
- save data per prompt, per layer, per position whenever the quantity is defined at that level
- only save a single scalar summary when the metric is truly aggregate-only
- do not collapse per-position data too early
- keep enough indexing to reconnect aggregate summaries back to row-level and tensor-level artifacts

Decoded output preservation
- always save top-k token ids
- always save decoded top-k token strings
- save source token id and decoded source token
- save target token id and decoded target token
- save final top-1 token id and decoded token
- save lens top-1 token id and decoded token
- if useful, save normalized decoded token forms used in surface metrics

Top-k structure / repetition support metrics
- repetition in top-k predictions for a single model/lens should be captured where meaningful
- examples:
  - repeated normalized token forms in top-k
  - repeated whitespace variants / punctuation variants in top-k
  - duplicate normalized forms ratio in top-k
- useful especially for fragmented tokenization and low-resource-language analysis
